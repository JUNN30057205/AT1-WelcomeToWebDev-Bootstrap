HTTP is an extensible protocol that relies on concepts like resources and Uniform Resource Identifiers (URIs), simple message structure, and client-server communication flow. Session management is the process of securing multiple requests to a service from the same user or entity. Session management manages sessions between the web application and the users. The communication between a web browser and a website is usually done over HTTP or HTTPS. When a user visits a website, a session is made containing multiple requests and responses over HTTP. In many cases, a session is initialized by authenticating a user or entity with factors such as a password. Once the user is authenticated, subsequent requests authenticate the session as opposed to the user themselves. Learn more.

Stateless programming is a paradigm in which the operations (functions, methods, procedures etc.) you implement are not sensitive to the state of the computation. Program does not maintain a state. This means that the program cannot take information about the last session into the next, such as settings the user chose or conditions that arose during the processing. Learn more.
